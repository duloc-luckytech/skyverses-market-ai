STREAMING MODE (aiChatStreamViaProxy):
──────────────────────────────────────

Request:
  POST /ai/chat HTTP/1.1
  Authorization: Bearer eyJhbGc...
  Content-Type: application/json
  
  {
    "messages": [
      {"role": "system", "content": "You are a helpful AI..."},
      {"role": "user", "content": "Help me with this..."}
    ],
    "stream": true,
    "max_tokens": 4096
  }

Response (SSE):
  HTTP/1.1 200 OK
  Content-Type: text/event-stream
  Cache-Control: no-cache
  
  data: {"choices":[{"delta":{"content":"Hello"}}]}
  data: {"choices":[{"delta":{"content":" there"}}]}
  data: {"choices":[{"delta":{"content":"!"}}]}
  data: [DONE]

JavaScript:
  let accumulated = '';
  await aiChatStreamViaProxy(
    messages,
    (token) => {
      accumulated += token;  // 'Hello', ' there', '!'
      setOutput(accumulated);
    }
  );
  // accumulated = 'Hello there!'

🔐 AUTHENTICATION & SECURITY
═══════════════════════════════════════════════════════════════════════════════

Frontend:
  - Token stored in localStorage as 'skyverses_auth_token'
  - Automatically included in getHeaders()
  - Sent as Authorization: Bearer {token}

Backend:
  - Middleware checks authentication before handling request
  - Rate limits per user ID (from token)
  - API keys managed server-side (NOT exposed to frontend)
  - Random key selection from active pool (listKeyGommoGenmini)

Upstream:
  - Backend holds actual EzAI API key
  - Frontend only has placeholder/test key
  - Server-side key rotation possible
